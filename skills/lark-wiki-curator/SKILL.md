# 飞书知识库管理

自动化的知识库整理和维护流程。

## 核心原则

- 知识不锁在本地，**沉淀到飞书知识库**
- 保持清晰的分类结构
- 定期检查和更新过时内容

## 知识库结构设计

```
知识空间（Space）
├── 📚 学习笔记/
│   ├── GLM Coding/
│   │   ├── 入门指南
│   │   ├── 工具链
│   │   ├── 官方最佳实践
│   │   └── 社区精选
│   ├── 技术调研/
│   └── 安全知识/
├── 📋 项目文档/
│   ├── nian-desktop/
│   ├── Team Dashboard/
│   └── 集团平台/
├── 📊 运维手册/
│   ├── OpenClaw 配置/
│   ├── 部署流程/
│   └── 应急预案/
└── 📰 每日简报/
    ├── 安全简报/
    └── 热榜速览/
```

## 操作流程

### 1. 初始化知识库

```
1. feishu_wiki_space list → 检查是否已有目标空间
2. 没有则 feishu_wiki_space create → 创建空间
3. 按结构创建目录节点
4. 记录 space_id 和各节点 token
```

### 2. 沉淀单个文档

```
输入: 本地 .md 文件 + 分类
流程:
  1. 确定目标父节点（从映射表查找）
  2. 读取本地文件
  3. feishu_wiki_space_node create → 创建子节点
  4. 更新同步映射
```

### 3. 批量沉淀

```
输入: 一组本地文件
流程:
  1. 扫描目标目录
  2. 按文件名/内容自动分类
  3. 批量创建（每批 3-5 个）
  4. 更新目录索引
```

### 4. 维护目录索引

在知识库根节点维护一个 `INDEX.md`（或飞书文档）：

```markdown
# 知识库目录

最后更新: YYYY-MM-DD

## 📚 学习笔记
- [GLM Coding 入门](链接) — 2026-04-05
- [GLM Coding 工具链](链接) — 2026-04-05
...

## 📋 项目文档
...
```

### 5. 定期巡检

建议 cron 每周一次：
- 检查本地新增文件是否已同步
- 检查飞书文档是否有过时标记
- 更新目录索引

## 分类规则

| 文件特征 | 分类 |
|---------|------|
| `*-study-*.md` / `*-学习*` | 学习笔记 |
| `*-project-*.md` / `*-项目*` | 项目文档 |
| `*-ops-*.md` / `*-运维*` / `*-deploy*` | 运维手册 |
| `*-briefing-*.md` / `安全简报*` | 每日简报 |
| 含 `API` / `接口` | 技术文档 |

## 映射文件

维护 `~/.openclaw/workspace/.wiki-map.json`：

```json
{
  "space_id": "wkspXXX",
  "nodes": {
    "学习笔记": { "token": "nodXXX", "path": "学习笔记" },
    "GLM Coding": { "token": "nodYYY", "parent": "学习笔记" }
  }
}
```
