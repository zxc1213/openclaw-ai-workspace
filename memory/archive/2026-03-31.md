## KCZX science-innovation-parent 环境变量整合

### 背景
- 开发环境：IDEA + envfile 插件加载 `.env`
- 生产环境：Docker 容器 + PM2 管理多服务
- `application.yml` 已用 `${VAR:default}` 语法读环境变量
- 但 Docker 里没有注入机制，PM2 JSON 也没传环境变量

### 改动清单（当前在默认分支执行）

#### 1. `shell/docker.sh`
- 新增 `ENV_FILE` 变量（默认 `env/.env.pro`），支持通过环境变量切换
- 新增 `load_env()` 函数：用 `while IFS='=' read` 逐行读取 .env 文件，`export` 到 shell 环境
- 在 `start()` 函数开头调用 `load_env`，PM2 子进程自动继承环境变量
- 找不到 .env 文件时给警告但不中断，降级到 application.yml 默认值

#### 2. `Dockerfile`
- 新增 `COPY /env/ /kczx/science-innovation-parent/env/` 把 env 目录打进去
- 新增 `ENV ENV_FILE=env/.env.pro` 默认加载 .env.pro
- 启动时可通过 `-e ENV_FILE=env/.env.ali` 切换环境

#### 3. `k8s/configmap.yaml`（新建，替代 4 个旧文件）
- 合并了：`common-configmap.yaml` + `communication-configmap.yaml` + `oss-configmap.yaml` + `homeSearch-configmap.yaml`
- 统一 name: `kczx-config`

#### 4. `k8s/secret.yaml`（新建，替代 3 个旧文件）
- 合并了：`common-secret.yaml` + `communication-secret.yaml` + `file-secret.yaml`
- 统一 name: `kczx-secrets`

#### 5. `k8s/gateway-deployment.yaml`
- envFrom 从 `kczx-common-config` 改为 `kczx-config` + `kczx-secrets`

#### 6. `k8s/services-deployment.yaml`
- 所有 12 个服务的 envFrom 统一改为 `kczx-config` + `kczx-secrets`

#### 7. 删除的旧文件
- `k8s/common-configmap.yaml`
- `k8s/common-secret.yaml`
- `k8s/communication-configmap.yaml`
- `k8s/communication-secret.yaml`
- `k8s/file-secret.yaml`
- `k8s/homeSearch-configmap.yaml`
- `k8s/oss-configmap.yaml`
