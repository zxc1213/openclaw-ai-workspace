## WSL2 WSLInterop 修复 + Windows Chrome 远程控制

### 问题
WSL2 无法执行 Windows .exe 文件（cmd.exe、powershell.exe 均报 `cannot execute binary file`），
WSLInterop binfmt_misc 条目缺失。

### 修复过程
1. `cat /proc/sys/fs/binfmt_misc/WSLInterop` → 文件不存在
2. 直接 `sudo tee /proc/sys/fs/binfmt_misc/WSLInterop` → Permission denied（条目不存在时不能直接写）
3. ✅ **解决方案**：`echo ':WSLInterop:M::MZ::/init:PF' | sudo tee /proc/sys/fs/binfmt_misc/register`
4. 验证：`cat /proc/sys/fs/binfmt_misc/WSLInterop` → 成功显示 interpreter /init flags: PF
5. 测试：`cmd.exe /c "echo hello from Windows"` → 输出正常

### Chrome 远程控制
- Chrome 启动命令：`cmd.exe /c start "" "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --remote-debugging-address=0.0.0.0 --user-data-dir="D:\Temp\chrome-profile-debug"`
- WSL2 通过 `172.25.192.1:9222` 访问 Windows Chrome 的 CDP 端口
- 现在可以从 WSL2 启动并控制 Windows Chrome

### 经验
- WSLInterop 条目缺失时，用 `/proc/sys/fs/binfmt_misc/register`（不是直接写 WSLInterop 文件）
- `cannot execute binary file` 不一定是架构问题，可能是 binfmt 条目缺失
- WSLInterop 修复后，之前"bb-browser 不可用"的问题理论上可以绕过

## Windows Chrome 远程控制 - 自动化配置完成

### 架构
- WSL → `172.25.192.1:9222` → Windows Chrome CDP
- Chrome 启动时默认绑定 `[::1]:9222`（IPv6 only）
- 通过 `netsh interface portproxy v4tov6` 将 IPv4 转发到 IPv6
- Windows 防火墙放行 9222 端口

### Windows 端持久化配置（已确认，重启不丢）
- **portproxy**: `0.0.0.0:9222 → [::1]:9222`（v4tov6）
- **防火墙**: 规则 "Chrome CDP 9222"，TCP 入站放行，所有配置文件

### WSL 端工具
- `scripts/chrome-remote.sh [url]` — 启动 Chrome / 打开新标签页
  - 通过 PowerShell Start-Process 启动（cmd.exe start 在 WSL bash 下超时）
  - 已运行则用 Target.createTarget 开新标签
- `scripts/cdp-control.py` — CDP 操作工具（screenshot/tabs/navigate/eval/click/type/keys/scroll/text/wait）

### Chrome 启动参数
- `--remote-debugging-port=9222`
- `--remote-allow-origins=*`
- `--user-data-dir=D:\Temp\chrome-profile-debug`（独立 profile，不影响日常 Chrome）

### 注意事项
- taskkill /IM chrome.exe 会杀掉所有 Chrome 进程（包括非 debug 的）
- 小红书搜索框 input 不响应 CDP 的 Input.insertText / dispatchKeyEvent，需用 Page.navigate 直接跳转搜索 URL
- Chrome CDP 只在 IPv6 localhost 监听，不能直接用 --remote-debugging-address=0.0.0.0（新版 Chrome 忽略）
