| 能力 | 实现方式 | 触发条件 |
|------|----------|----------|

| **结构化沉淀** | Daily Notes → heartbeat-reflections/ 去重写入 | 每次心跳 |
| **MEMORY.md 定期精简** | 过时条目移到 archive，合并重复 | 手动（每月） |

### 还没完全自动化的

1. **MEMORY.md 自动精简**——现在是手动，可以加个 cron 定期跑
2. **heartbeat-reflections 去重的准确度**——靠 LLM 判断"是否高度相似"，偶尔会漏或误判
3. **Daily Notes 自动摘要**——7 天后归档时可以先生成摘要再归档，方便后续语义搜索
4. **archive 目录的二次清理**——archive 也会膨胀，需要定期清除真正无价值的旧文件

### 核心原理

其实不复杂，就是三步循环：

```
心跳触发 → 扫描变化 → 按规则处理（归档/去重/精简/索引更新）
```

关键是**规则要写死在 HEARTBEAT.md 里**，让 agent 每次心跳都执行同样的流程，不需要人工干预。
