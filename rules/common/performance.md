# Performance Rules

## 模型选择策略

| 场景 | 推荐模型 | 原因 |
|------|----------|------|
| 主开发、编排、复杂编码 | GLM-5 / GLM-5.1 | 最强编码能力 |
| 轻量 worker、快速任务 | GLM-Turbo / Flash | 速度快、成本低 |
| 复杂架构决策、分析 | GLM-5 | 最强推理 |

## 上下文管理

- 大规模重构/多文件功能：注意上下文窗口消耗
- 简单编辑/独立工具/文档更新：无压力
- 复杂任务优先用 subagent（独立上下文）

## Subagent 编排

- 独立任务 → **并行** spawn 多个 subagent
- 依赖任务 → 串行，上游完成再启动下游
- 长任务 → 后台运行，主会话保持响应
