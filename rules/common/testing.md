# Testing Rules

## 覆盖率要求

- 核心逻辑：80%+ 覆盖率
- 工具函数/公共 API：必须有测试
- UI/样式代码：可选

## 测试类型

| 类型 | 范围 | 优先级 |
|------|------|--------|
| 单元测试 | 函数、工具、组件 | 必须 |
| 集成测试 | API、数据库操作 | 重要 |
| E2E 测试 | 关键用户流程 | 按需 |

## TDD 流程（适用于复杂功能）

1. **RED** — 先写失败的测试
2. **GREEN** — 写最小实现让测试通过
3. **REFACTOR** — 重构
4. 验证覆盖率

## 修复测试失败

1. 先检查测试本身是否正确
2. 检查 mock 是否匹配
3. 修实现，不修测试（除非测试写错了）
